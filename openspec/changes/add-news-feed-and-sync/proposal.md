## Why

The Event Log tab in port and chokepoint detail views only surfaces internal `RiskStoryEvent` anomalies generated by the scoring pipeline. Operators want real-world context — current Google News coverage of geopolitical, weather, labor, or congestion events affecting the entity — alongside the system-detected anomalies. Additionally, while data collectors run on a Celery-beat schedule (PortWatch hourly, others daily), there is no UI affordance to force-fetch on demand: users have to wait for the next scheduled tick or invoke the bearer-protected `POST /sync/{source}` endpoint manually.

## What Changes

- Add a Google News collector that fetches recent news items per port and chokepoint (keyword = entity name + aliases) and persists them to a new `news_item` table keyed by URL+entity.
- Add a Celery task `collect.news` and schedule it every 6 hours (off-peak: :15 past the hour every 6h) plus include it in the `collect_all` chord.
- Expose a new endpoint `GET /entities/{type}/{id}/news` returning recent news items (paginated, default 20, max 100), ordered by `published_at` desc.
- Extend the Event Log tab on `PortDetailView` and `ChokepointDetailView` to render two grouped sections: **System events** (existing `/story` data) and **News** (new endpoint). News rows show title, source, published time, and link out.
- Add a **Sync** button to the page header on port/chokepoint detail views (and optionally the dashboard header) that calls `POST /sync/{source}` for the relevant sources and shows task progress via toast / polling.

## Capabilities

### New Capabilities
- `news-feed`: Fetch, persist, and serve Google News items per port/chokepoint for the Event Log tab.
- `manual-sync`: User-initiated force-fetch trigger from the UI that invokes collector tasks and surfaces task status.

## Impact

- **Backend new code**: `app/collectors/google_news.py`, `app/api/routes/news.py`, `app/schemas/news.py`, new model `NewsItem` in `app/db/models.py`, Alembic migration, new Celery task in `app/tasks/collect.py`, schedule entry in `app/tasks/schedule.py`.
- **Backend modified**: `app/api/router.py` (register news route), `app/api/routes/sync.py` (include `news` in `_VALID_SOURCES` and task map), `collect_all` chord (add `collect_news.s()`).
- **Frontend new code**: `frontend/src/api/news.ts`, `SyncButton` component, news subsection in `EventLog` (used by both `PortDetailView` and `ChokepointDetailView`).
- **Frontend modified**: `PortDetailView.tsx`, `ChokepointDetailView.tsx`, `frontend/src/api/types.ts` (NewsItem type), `frontend/src/api/sync.ts` (new wrapper).
- **Dependencies**: `feedparser` (Python) to parse Google News RSS — no API key required. Fallback / rate-limit handling per Google News RSS terms.
- **Config**: optional `NEWS_FETCH_ENABLED`, `NEWS_MAX_ITEMS_PER_ENTITY` env vars (sensible defaults).
- **Auth**: news endpoint public read (same as `/story`); sync endpoint stays bearer-protected — frontend reads token from settings/env, same flow as existing admin actions.
